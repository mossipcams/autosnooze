Create `src/components/autosnooze-active-pauses.ts` as a LitElement class named `AutoSnoozeActivePauses`.

**Properties (reactive, passed by parent):**
- `hass?: HomeAssistant` -- `@property({ attribute: false })` -- the HA state object
- `pauseGroups: PauseGroup[]` -- `@property({ attribute: false })` -- grouped paused automations (default `[]`)
- `pausedCount: number` -- `@property({ type: Number })` -- total count of paused automations (default `0`)

**Internal state:**
- `_wakeAllPending: boolean` -- `@state()` -- two-tap confirmation state (default `false`)

**Private fields (not reactive):**
- `_wakeAllTimeout: number | null` -- timeout ID for wake-all confirmation reset
- `_interval: number | null` -- countdown interval ID
- `_syncTimeout: number | null` -- sync timeout ID for second-boundary alignment

**Static styles:**
- `static styles = activePausesStyles;` (import from `../styles/active-pauses.styles.js`)

**Lifecycle:**
- `connectedCallback()`: Call `super.connectedCallback()`, clear any existing interval/syncTimeout, then call `_startSynchronizedCountdown()`. Copy the exact pattern from `autosnooze-card.ts` lines 195-207 (the interval/syncTimeout clearing + start logic).
- `disconnectedCallback()`: Call `super.disconnectedCallback()`, clear `_interval`, `_syncTimeout`, `_wakeAllTimeout` (copy pattern from `autosnooze-card.ts` lines 215-231, but only the three timers this component owns -- NOT searchTimeout, toastTimeout, toastFadeTimeout which stay in parent).

**Timer methods (move from parent):**
- `_startSynchronizedCountdown()`: Exact copy from `autosnooze-card.ts` lines 242-254.
- `_updateCountdownIfNeeded()`: Simplified version -- if `this.pauseGroups.length > 0`, call `this.requestUpdate()`. (The parent version checks `getPaused(this.hass)`, but the child already has `pauseGroups` as a property, so just check that.)

**Render method:**
- Copy the template from `_renderActivePauses()` (lines 1105-1155 of `autosnooze-card.ts`) into the `render()` method.
- Replace `this._getPausedGroupedByResumeTime()` with `this.pauseGroups` (data comes via property).
- Replace `this._formatDateTime(...)` with `formatDateTime(...)` using the imported utility directly. Pass `this.hass?.locale?.language` as the locale argument.
- Replace `this._formatCountdown(...)` with `formatCountdown(...)` using the imported utility directly.
- Replace `this._wake(auto.entity_id)` with `this._fireWake(auto.entity_id)`.
- Replace `this._handleWakeAll()` with `this._handleWakeAll()` (kept locally but fires event instead of calling service).
- The `if (pausedCount === 0) return html\`\`;` guard uses `this.pausedCount`.

**Event methods:**
- `_fireWake(entityId: string)`: Dispatch `new CustomEvent('wake-automation', { detail: { entityId }, bubbles: true, composed: true })`.
- `_fireWakeAll()`: Dispatch `new CustomEvent('wake-all', { bubbles: true, composed: true })`.

**Wake-all confirmation logic:**
- `_handleWakeAll()`: Copy the logic from `autosnooze-card.ts` lines 772-800, BUT replace the actual service calls and toast with event dispatching:
  - When `_wakeAllPending` is true (second tap): clear timeout, set `_wakeAllPending = false`, call `this._fireWakeAll()`.
  - When `_wakeAllPending` is false (first tap): dispatch haptic feedback `hapticFeedback('medium')`, set `_wakeAllPending = true`, set timeout to reset after `UI_TIMING.WAKE_ALL_CONFIRM_MS`.
