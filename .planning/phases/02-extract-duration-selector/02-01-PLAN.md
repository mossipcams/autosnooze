**Properties (reactive, passed by parent):**

- `scheduleMode: boolean` -- `@property({ type: Boolean })` -- whether schedule mode is active (default `false`)
- `customDuration: ParsedDuration` -- `@property({ attribute: false })` -- current duration object (default `{ days: 0, hours: 0, minutes: 30 }`)
- `customDurationInput: string` -- `@property({ type: String })` -- current custom input text (default `'30m'`)
- `showCustomInput: boolean` -- `@property({ type: Boolean })` -- whether custom input is visible (default `false`)
- `lastDuration: LastDurationData | null` -- `@property({ attribute: false })` -- last used duration (default `null`)
- `disableAtDate: string` -- `@property({ type: String })` -- schedule mode disable date (default `''`)
- `disableAtTime: string` -- `@property({ type: String })` -- schedule mode disable time (default `''`)
- `resumeAtDate: string` -- `@property({ type: String })` -- schedule mode resume date (default `''`)
- `resumeAtTime: string` -- `@property({ type: String })` -- schedule mode resume time (default `''`)

**Static styles:**
- `static styles = durationSelectorStyles;` (import from `../styles/duration-selector.styles.js`)

**Helper methods (internal, not events):**

1. `_getDurationPills(): { label: string; minutes: number | null }[]`
   - Copy exact logic from `autosnooze-card.ts` lines 764-782
   - Reads `this.hass?.states?.['sensor.autosnooze_snoozed_automations']?.attributes?.duration_presets`
   - Falls back to `DEFAULT_DURATIONS.filter(d => d.minutes !== null)`
   - Always appends `{ label: 'Custom', minutes: null }` at the end
   - Returns the array of pill objects

2. `_renderLastDurationBadge(): TemplateResult | string`
   - Copy exact logic from `autosnooze-card.ts` lines 784-819
   - Returns early with `''` if `!this.lastDuration`
   - Computes `basePresets` same as `_getDurationPills()` but filters out Custom
   - Checks if `lastMinutes` is unique from presets using `!basePresets.some(d => d.minutes === lastMinutes)`
   - Returns `''` if not unique
   - Otherwise renders the badge button with `formatDurationShort`, `localize`, click handler that fires `duration-change` event
   - Uses `durationToMinutes(this.customDuration)` to compute `currentMinutes` for `isActive` check
   - The `isActive` check is: `!this.showCustomInput && lastMinutes === currentMinutes`

3. `_renderDateOptions(): TemplateResult[]`
   - Copy from `autosnooze-card.ts` lines 757-762
   - Uses `generateDateOptions(365, this.hass?.locale?.language)`
   - Maps each option to `html\`<option value="${opt.value}">${opt.label}</option>\``

4. `_getDurationPreview(): string`
   - Copy from `autosnooze-card.ts` lines 419-423
   - Parses `this.customDurationInput` using `parseDurationInput`
   - If parsed, returns `formatDuration(parsed.days, parsed.hours, parsed.minutes)`
   - Otherwise returns `''`

5. `_isDurationValid(): boolean`
   - Copy from `autosnooze-card.ts` lines 425-427
   - Returns `isDurationValid(this.customDurationInput)`

**Event methods:**

1. `_fireDurationChange(minutes: number, options?: { showCustomInput?: boolean }): void`
   - Converts `minutes` to `ParsedDuration` using `minutesToDuration(minutes)`
   - Formats duration using `formatDurationShort` to create input string
   - Dispatches `CustomEvent('duration-change', { detail: { minutes, duration, input, showCustomInput }, bubbles: true, composed: true })`
   - The `showCustomInput` flag in detail defaults to `options?.showCustomInput ?? false`
   - The `input` field is the formatted short string

2. `_fireCustomDurationChange(value: string): void`
   - Parses `value` using `parseDurationInput(value)`
   - If parsed, computes `totalMinutes = durationToMinutes(parsed)`
   - Dispatches `CustomEvent('duration-change', { detail: { minutes: totalMinutes, duration: parsed, input: value }, bubbles: true, composed: true })`

3. `_fireScheduleModeChange(enabled: boolean): void`
   - Dispatches `CustomEvent('schedule-mode-change', { detail: { enabled }, bubbles: true, composed: true })`

4. `_fireScheduleFieldChange(field: string, value: string): void`
   - Dispatches `CustomEvent('schedule-field-change', { detail: { field, value }, bubbles: true, composed: true })`
